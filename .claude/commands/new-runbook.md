`$ARGUMENTS`를 파싱하여 운영 Runbook을 생성합니다.
- 첫 번째 인자: 대상 서비스 (ec2, eks, rds, vpc 등)
- 나머지 인자: 작업명

## 파일 생성 규칙

1. 파일명: `runbook-{서비스}-{작업명}.md`
2. 저장 위치: `docs/{해당 카테고리}/`
3. `templates/runbook.md` 템플릿 기반으로 작성

## 작성 시 필수 포함 사항

- **사전 체크리스트**: 최소 4개 항목
- **환경 변수 설정**: `export` 형식으로 명시
- **단계별 명령어**: 복붙 즉시 실행 가능
- **확인 명령어**: 각 스텝 완료 후 검증 방법
- **롤백 절차**: 되돌릴 수 있는 명령어 포함
- **모니터링 포인트**: 작업 후 감시할 CloudWatch 지표

## 작업 특성별 추가 요소

| 작업 유형 | 추가 항목 |
|----------|---------|
| 유지보수 (PM) | 사전 공지 템플릿, Connection Draining |
| 마이그레이션 | Before/After 비교, 데이터 정합성 확인 |
| 스케일 변경 | 용량 계획, 비용 영향 |
| 보안 패치 | CVE 번호, 패치 내용 요약 |
